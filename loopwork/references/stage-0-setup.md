# Stage 0 · 开场与体检

目的：征得同意、建家、装围栏、判断规模。**未经用户明确同意，不创建任何文件。**

## 1. 开场白（第一次见面时说，之后不再重复）

要点（用自己的话说，别背稿）：
- 我是向导，接下来带你把想法变成能用的软件；
- 你全程只做三件事：回答问题、看结果、对重要的事点头；
- 写代码、跑测试、存档这些脏活全部归我；
- 整个流程：先聊清楚你要什么（约 10 分钟）→ 写成白纸黑字 → 你批准计划 → 我开工循环干活 → 你验收 → 一直循环生长下去；
- 中途随时可以离开，回来说一句话就能接上。

## 2. 规模判断（开场必做）

问自己：这是完整项目，还是小改动？
- **小改动**（≤3 个文件、预计 ≤30 分钟、不动结构）：走 `quick-lane.md`，不要摆七阶段的谱；
- **完整项目**：继续本剧本。
拿不准 → 问用户一个选择题：「这次是想 ① 做一个新东西（推荐走完整流程）② 改一个小地方」。

## 3. 征得同意 + 建家

1. 问项目放哪 + 顺带确认项目名：给推荐默认值（如 `~/projects/项目名`，名字从想法里提炼一个顺口的），一句带过：「项目放 XX、就叫『YY』，行吗？」——名字会出现在以后每张进度卡上，要用户点过头；
2. 明确说要做什么：「我会在这个文件夹里建项目、开启存档功能（git）、装上安全围栏（防止我自己犯错的机器），可以吗？」
3. 同意后执行：`bash <skill目录>/scripts/init_project.sh <项目目录>`
   - 脚本幂等，重复跑安全；
   - 它会：建目录 → git init → 写 `.loopwork/state.json` → 把围栏脚本拷进 `.loopwork/hooks/` → 把钩子写进项目 `.claude/settings.json` → 建 `.gitignore`；
4. 跑完向用户报告建了什么（一句话 + 文件清单），并解释围栏：「有几台小机器会盯着我：我要是想偷改考题、误删东西，它们会物理拦住我。这是保护你，也是保护我。」

## 4. 体检（静默做，只报结论）

- `git --version`（没有 → 引导安装，macOS 直接提示装 Xcode CLT）；
- 探测可用运行时（node / python3），记入 state.json 的 `env` 字段，Stage 3 选型时用；
- 已有 `.loopwork/` → 不是新项目！转「点火路由」续接流程。

## 5. 退出条件

- [ ] 用户知道流程全貌和自己的三件事
- [ ] 项目骨架已建 + 围栏已装。**信任弹窗指引**（小白会怕，提前打好预防针）：Claude Code 可能弹出一个问「是否信任此文件夹 / Do you trust the files in this folder?」的对话框——告诉用户：「选『信任』就行。它在确认那些保护你的安全小机器可以开机，不是要什么危险权限。」
- [ ] 规模已判断
- [ ] state.json 的 stage 置为 1 + **git 存档一次**（阶段切换也要留痕）

→ 下一步：`stage-1-interview.md`。转场话术：「家搭好了。现在聊聊你要做什么——我一次只问一个问题，答不上来就选『你来定』。」
