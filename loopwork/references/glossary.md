# 白话词典（完整版）

任何黑话首次出现，用这里的比喻解释。解释格式：是什么 + 为什么需要它。

| 白话词 | 实际概念 | 是什么 | 为什么需要它 |
|---|---|---|---|
| 向导 | 本 skill | 带你走完全程的老师傅 | 你不需要懂流程，它需要 |
| 考题 | 自动化测试 | 一批自动判分的题：代码对，题就过 | 你看不懂代码，但考题能替你盯着「它还对不对」；以后每次改动都自动重考一遍 |
| 先红后绿 | TDD | 先让考题失败（红），再写代码让它通过（绿） | 一张永远给满分的考卷保护不了你——先亲眼看它「红」，才证明考卷是真的 |
| 存档 | git commit | 拍快照存进历史，随时能回到这一刻 | 游戏存档的意思：改坏了也不怕 |
| 存档点 | git 历史 | 所有快照的清单 | 灾难恢复的后悔药 |
| 围栏 | hooks（守护脚本） | 盯着 AI 的小机器：想偷改考题、误删东西会被物理拦住 | 规矩写在纸上会被忘，装在机器里才可靠 |
| 项目规矩 | constitution / rules.md | 本项目绝不许做的事，白纸黑字 | 红线要立在开工前，不是出事后 |
| 任务清单 | tasks.md | 「下一步干啥」的清单，做完一条勾一条 | 你和 AI 共用的进度表 |
| 问题本 | BLOCKED.md | 要你拍板的事都记在这，AI 记完就跳过继续干 | 你不在场时机器不停工，你回来一把清算 |
| 验收单 | 手工验收脚本 | 照着点一遍就能确认功能是真的 | 不用读代码也能验收 |
| 判卷员 | 只读 code reviewer | 只能看不能改的检查员 | 写代码的和判卷的不能是同一个脑子 |
| 跑批 | 循环执行一批任务 | AI 一条接一条自动干完这一批 | 你挂机，它干活 |
| 点火 | 发起新一轮工作 | 回来说一句话，机器就转起来 | 「继续我的记账项目」就是钥匙 |
| 一圈 | 一个外循环周期 | 点火→跑批→收货→续单 | 项目生长的基本节拍 |
| 续单 | gap-check | 清单空了，盘点缺口列出下一批 | 清单空 ≠ 做完，只是该加燃料了 |
| 内循环/外循环 | 任务级/项目级循环 | AI 在内圈干活，你在外圈掌舵 | 执行自主，方向人定 |
| 进环仪式 | 首航→循环模式切换 | 第一版完成时的换挡仪式 | 从「跟着走」变成「你掌舵」 |
| 挂机档 | 批模式 / /goal | 做完一条自动接下一条，直到批空才来找你 | 连点火都省了 |
| 里程碑 | milestone | 值得庆祝的节点：首次跑通、过半、首航完成、首次独立转一圈 | 长路要有奖励，不然坚持不下来 |
