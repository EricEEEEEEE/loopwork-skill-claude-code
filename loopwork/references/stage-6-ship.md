# Stage 6 · 首航收尾与上线护送

目的：跨过「代码写完」到「用得上」的断层，然后举行进环仪式。**「只自己用」是完全合法的终点，绝不推销部署。**

## 1. 先问归宿（选择题）

「第一版完成了。它的归宿是：① 就在这台电脑上自己用（推荐，零成本零风险）② 让家人朋友也能访问 ③ 正式放到网上」

## 2. 三份收尾产物（无论选哪个都要做）

- `README.md`：这项目是什么、能干什么（给未来的自己和访客）；
- `使用说明.md`：怎么启动、怎么日常使用、怎么关（含启动命令，照抄可用）；
- `维护手册.md`：出问题怎么办——附三句现成求助话术：
  1. 「继续我的 {项目名} 项目」（日常点火）
  2. 「我的 {项目名} 打不开了，帮我看看」（急救）
  3. 「给 {项目名} 加个 XX 功能」（生长）

## 3. 上线护送（仅当用户选 ②③）

铁律：**每一步单独护送、单独确认；凡花钱的选项显式标 💰 并给免费替代；账号注册、按钮点击由用户亲手做，你只出「照着点」的分步说明。**

- 优先推免费静态托管路径；需要账号 → 告诉用户去哪注册、注册完回来说一声；
- **密钥纪律**：需要任何 key/token 时，引导用户放进环境配置文件（已在 .gitignore），你永不经手明文——让用户自己粘贴进指定文件，你只验证「文件存在且非空」；用户误把密钥粘进对话 → 立即提醒撤回，绝不写入任何文件；
- 每步做完让用户回报看到了什么，对不上 → 停下排查，不带病前进。

## 4. 进环仪式（上半场真正的终点）

1. 总结：从想法到现在，做了什么、用户拥有了什么（点名每个功能）；
2. 宣告换挡：「从现在起我们换一种合作方式：**你出方向，我跑圈**。想加功能、发现毛病、或只是想看看它还能长成什么样——回来说一句话就是点火。」
3. 三句话术卡（见上）再发一遍；
4. `progress.py set stage loop` + `bump-cycle`；庆祝 `voyage_done`；git 存档。

自此项目进入下半场，之后每次激活走 `loop-mode.md`。
